# 常见问题

# 如果显示种子库数量为0，只有这几种情况：1.种子库没映射进容器 2.下载器配置页面种子目录填写的不对（public前面不要反斜杠） 3.下载器没有启用，我求求你们好好看教程

# 如果一直显示在执行种子库计算任务（好几个小时了），请先在终止计划任务，然后清理一下任务锁，然后再次手动执行一次如果还是这样，请加群：922047668，来骂我！

# 如果Docker日志中类似这样显示的，请你按照Docker如何安装那页的教程仔细看看映射目录，100个人里面有99.5个是没看清楚的

# 如果计划任务日志中这样显示，说明你下载器删除了，但是你计划任务中还是没更新，你都删了前任了，还想要继续约是不是不道德啊！

# 如果计划任务页面跳出这个提示的，你绝对是表达式没写好，然后作者我又没做校验，然后提交给程序，程序就懵逼了啊，你这tm我怎么执行啊，然后这句话翻译成英语就是json_decode()巴拉巴拉

# 如果用的是Qb和Tr套件的话，八成目录：

套件版transmission种子保存路径 /volume1/@appdata/transmission/torrents 套件版qBittorrent种子保存路径 /volume1/@appdata/qBittorrent/qBittorrent_conf/data/BT_backup
